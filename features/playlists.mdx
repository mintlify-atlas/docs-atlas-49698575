## Overview

Playlists in Swar allow users to organize their favorite songs into custom collections. Users can create multiple playlists, add or remove songs, and control privacy settings.

## Managing Playlists

### Viewing Your Playlists

The Library page displays all user-created playlists along with the default "Liked Songs" collection.

```jsx Library Component (Library.jsx:73-109)
const Library = () => {
  const [data, setData] = useState({
    playlists: [],
    loading: true,
    error: null,
  });

  const defaultPlaylist = {
    publicId: "0",
    playlistName: "Liked Songs",
  };

  return (
    <div className="space-y-4">
      <PlaylistCard

        onUpdate={fetchPlaylists}
      />
      {playlists.map((playlist) => (
        <PlaylistCard
          key={playlist.playlistId}
          playlist={playlist}
          onUpdate={fetchPlaylists}

  );
};
```

**API Endpoint:**
```http
GET /api/v1/Playlist/GetAllPlaylistsByUserId
Authorization: Bearer {accessToken}
```

### Creating a Playlist

Users can create new playlists with custom names, descriptions, and privacy settings.

<Steps>
  <Step title="Open Playlist Modal">
    Click the "+" button in the Library page to open the playlist creation modal.
  </Step>
  
  <Step title="Enter Playlist Details">
    Provide a playlist name (required) and optional description. Choose whether to make the playlist private.
  </Step>
  
  <Step title="Create Playlist">
    Click "Add" to create the playlist. The new playlist appears in your library immediately.
  </Step>
</Steps>

```jsx Create Playlist Handler (Library.jsx:55-71)
const handleCreate = async () => {
  setIsAdding(true);
  try {
    await swarApiClient.post("Playlist/CreatePlaylist", {
      playlistName: newPlaylist.playlistName,
      description: newPlaylist.description,
      isPrivate: newPlaylist.isPrivate,
    });
    fetchPlaylists();
    toast.success("Playlist created successfully.");
  } catch {
    toast.error("An error occurred while creating the playlist.");
  } finally {
    setIsAdding(false);
    onClose();
  }
};
```

**API Endpoint:**
```http
POST /api/v1/Playlist/CreatePlaylist
Authorization: Bearer {accessToken}
```

<Note>
Playlist names are required and cannot be empty. The system will show a validation error if you try to create a playlist without a name.
</Note>

### Updating Privacy Settings

Users can toggle playlist privacy between public and private.

**API Endpoint:**
```http
PUT /api/v1/Playlist/UpdatePlaylistPrivacy/{playlistId}
Authorization: Bearer {accessToken}
```

<CodeGroup>
```typescript Request Body
{
  "isPrivate": true
}
```
</CodeGroup>

<Tip>
Private playlists are only visible to you, while public playlists can be shared with others.
</Tip>

### Deleting a Playlist

Users can permanently delete playlists they own.

**API Endpoint:**
```http
DELETE /api/v1/Playlist/DeletePlaylist/{playlistId}
Authorization: Bearer {accessToken}
```

<Warning>
Deleting a playlist is permanent and removes all songs from that playlist. This action cannot be undone.
</Warning>

## Managing Playlist Songs

### Adding Songs to a Playlist

Users can add songs to any of their playlists from the player or search results.

```jsx Add to Playlist Hook (usePlaylistActionsBase.js:22-35)
const handleAddToPlaylist = useCallback(
  async (playlistId, songId) => {

          songId,

    );
  },
  [handleApiAction, swarApiClient]
```

**API Endpoint:**
```http
POST /api/v1/PlaylistSongs/AddSongToPlaylist
Authorization: Bearer {accessToken}
```

<CodeGroup>

<Note>

</Note>

### Removing Songs from a Playlist

Users can remove individual songs from playlists using the dropdown menu.

```jsx Remove Song Handler (Playlist.jsx:36-46)
const handleRemoveSong = async (songId) => {
  try {
    await handleRemoveFromPlaylist(playlistInfo.playlistId, songId);
    const updatedSongs = songs.filter((song) => song.id !== songId);
    setSongs(updatedSongs);
    playlistInfo.songsCount -= 1;
  } catch {
    setSongs(playlistSongs);
    playlistInfo.songsCount += 1;
  }
};
```

**API Endpoint:**
```http
DELETE /api/v1/PlaylistSongs/RemoveSongFromPlaylist/{playlistId}/{songId}
Authorization: Bearer {accessToken}
```

### Playlist Card

The `PlaylistCard` component displays individual playlists in the library with cover art and metadata.

<Tip>
Playlists with 4 or more songs display a grid of the first 4 song covers. Playlists with fewer songs show a single cover image.
</Tip>

## Error Handling

The playlist feature includes comprehensive error handling:

- **404 Not Found**: Playlist doesn't exist or has been deleted
- **403 Forbidden**: User doesn't have permission to access the playlist
- **409 Conflict**: Song already exists in the playlist
- **500 Internal Server Error**: Server-side error occurred

