## Overview

The Search feature allows users to find songs, artists, and albums quickly. Search results display relevant tracks along with extracted artist information and trending songs.

## Search Interface

### Search Bar Component

The search bar appears in the navigation on desktop and at the top of the Search page on mobile.

```jsx Search Page (Search.jsx:42-48)
const Search = () => {
  const [searchParams] = useSearchParams();
  const query = searchParams.get("q") || "";

  return (
    <div className="block mb-6 md:hidden">
      <SearchBar />
    </div>
  );
};
```

### Initiating a Search

Users enter a search query in the search bar, which updates the URL with the query parameter:

```
https://swar.app/search?q=shape%20of%20you
```

## Search Flow

<Steps>
  <Step title="Enter Query">
    User types a search term in the search bar (e.g., "shape of you").
  </Step>
  
  <Step title="Navigate to Search Page">
    The app navigates to `/search?q=shape%20of%20you` with the query parameter.
  </Step>
  
  <Step title="Fetch Results">
    The `useSearchData` hook automatically fetches results based on the query.
  </Step>
  
  <Step title="Display Results">
    Songs, artists, and trending tracks are displayed in separate sections.
  </Step>
</Steps>

## Search Implementation

### Search Data Hook

The `useSearchData` hook handles fetching and processing search results:

```jsx useSearchData Hook (useSearchData.js:11-41)
const useSearchData = (query) => {
  const [searchResults, setSearchResults] = useState([]);
  const [artists, setArtists] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const fetchSearchData = useCallback(async () => {
    if (!query) {
      setError({ message: "No search query provided.", status: 404 });
      return;
    }

    setLoading(true);
    setError(null);

    try {
      const { data: searchData } = await songApiClient.get(
        `SongsData/GetSongsByQuery?query=${query}&lyrics=false&songData=true`
      );

      const artistSet = new Set(
        searchData.flatMap((song) => song.primary_artists)
      );
      setArtists(Array.from(artistSet));
      setSearchResults(searchData);
    } catch (ex) {
      setError({
        message:
          ex.response?.status === 404
            ? "No results found, try a different search term."
            : "Oops! Something went wrong.",
        status: ex.response?.status || 500,
      });
    } finally {
      setLoading(false);
    }
  }, [query, songApiClient]);

  useEffect(() => {
    fetchSearchData();
  }, [fetchSearchData]);

  return { searchResults, artists, loading, error };
};
```

**API Endpoint:**
```http
GET /api/v1/SongsData/GetSongsByQuery?query={query}&lyrics=false&songData=true
Authorization: Bearer {accessToken}
```

### Query Parameters

- `query`: The search term (required)
- `lyrics`: Whether to include lyrics in results (default: `false`)
- `songData`: Whether to include full song data (default: `true`)

The Search page displays results in three distinct sections:

### 1. Search Results Section

Displays songs matching the search query:


## Reusable Section Renderer

The search page uses a reusable function to render each section consistently:

```jsx Section Renderer (Search.jsx:23-40)
const renderSection = (
  title,
  subtitle,
  isLoading,
  loadingComponent,
  items,
  renderItem
) => (
  <div className="grid gap-5">
    <div className="-mb-[3px]">
      <h1 className="text-lg font-bold">{title}</h1>
      <p className="mb-2 text-xs">{subtitle}</p>
    </div>
    <div className="flex items-center gap-6 pb-4 mb-3 overflow-x-auto">
      {isLoading ? loadingComponent : items.map(renderItem)}
    </div>
  </div>
);
```

## Performance Considerations

### Automatic Re-fetching

The search automatically re-fetches when the query parameter changes:

```jsx Auto-fetch on Query Change (useSearchData.js:43-45)

  fetchSearchData();
}, [fetchSearchData]);
```

### Empty State Handling

If search returns no results, appropriate empty states are displayed rather than showing errors.

## Integration with Other Features

### Navigation to Player

Clicking any song from search results navigates to the player:

```jsx
const navigate = useNavigate();
const handleSongClick = (songId) => {
  navigate("/player", { state: { songId } });
};
```

### Adding to Playlists

From search results, users can add songs to playlists using the playlist actions.

### Liking Songs

Songs in search results can be liked directly, adding them to the Liked Songs collection.
