## Get Current User

<RequestExample>
```bash GET /api/v1/users/me
curl -X GET https://api.swar.com/api/v1/users/me \
  -H "Authorization: Bearer {accessToken}"
```
</RequestExample>

Retrieve the authenticated user's profile information.

<Note>
  Requires authentication with Admin or User role.
</Note>

### Response

<ResponseField name="userId" type="integer">
  Unique user identifier
</ResponseField>

<ResponseField name="externalId" type="string">
  External provider ID if applicable
</ResponseField>

<ResponseField name="name" type="string">
  User's full name
</ResponseField>

<ResponseField name="email" type="string">
  User's email address
</ResponseField>

<ResponseField name="role" type="string">
  User's role
</ResponseField>

<ResponseField name="status" type="string">
  Account status
</ResponseField>

<ResponseField name="registrationDate" type="string">
  ISO 8601 formatted registration timestamp
</ResponseField>

<ResponseExample>
```json Response
{
  "userId": 123,
  "externalId": null,
  "name": "John Doe",
  "email": "john@example.com",
  "role": "User",
  "status": "Active",
  "registrationDate": "2026-03-03T10:30:00Z"
}
```
</ResponseExample>

### Error Codes

- `404` - User not found
- `500` - Internal server error

## Update Current User

<RequestExample>
```bash PATCH /api/v1/users/me
curl -X PATCH https://api.swar.com/api/v1/users/me \
  -H "Authorization: Bearer {accessToken}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Smith",
    "email": "johnsmith@example.com",
    "gender": "Male"
  }'
```
</RequestExample>

Update the authenticated user's profile information.

<Note>
  Requires authentication with Admin or User role. All fields are optional.
</Note>

### Request Body

<ParamField body="name" type="string">
  User's full name. Maximum 100 characters.
</ParamField>

<ParamField body="email" type="string">
  User's email address. Must be valid email format, maximum 255 characters.
</ParamField>

<ParamField body="gender" type="string">
  User's gender. Must be one of: "Male", "Female", "Other". Maximum 10 characters.
</ParamField>

### Response

Returns the updated user object with the same structure as Get Current User.

### Error Codes

- `400` - Invalid request or validation error
- `401` - Inactive account
- `404` - User not found
- `409` - Email already exists
- `500` - Internal server error

## Update Password

<RequestExample>
```bash PATCH /api/v1/users/me/password
curl -X PATCH https://api.swar.com/api/v1/users/me/password \
  -H "Authorization: Bearer {accessToken}" \
  -H "Content-Type: application/json" \
  -d '{
    "oldPassword": "CurrentPass123",
    "newPassword": "NewSecurePass456"
  }'
```
</RequestExample>

Update the authenticated user's password.

<Note>
  Requires authentication with Admin or User role.
</Note>

### Request Body

<ParamField body="oldPassword" type="string" required>
  Current password for verification
</ParamField>

<ParamField body="newPassword" type="string" required>
  New password to set
</ParamField>

### Response

Returns the updated user object with the same structure as Get Current User.

### Error Codes

- `400` - Invalid request or weak password
- `401` - Invalid old password or inactive account
- `404` - User not found
- `500` - Internal server error

## Delete Current User

<RequestExample>
```bash DELETE /api/v1/users/me
curl -X DELETE https://api.swar.com/api/v1/users/me \
  -H "Authorization: Bearer {accessToken}"
```
</RequestExample>

Delete the authenticated user's account.

<Warning>
  This action cannot be undone. All user data including playlists, liked songs, and play history will be removed.
</Warning>

<Note>
  Requires authentication with Admin or User role.
</Note>

### Response

Returns the deleted user object.

### Error Codes

- `404` - User not found
- `500` - Internal server error

## Deactivate User

<RequestExample>
```bash POST /api/v1/users/{id}/deactivate
curl -X POST https://api.swar.com/api/v1/users/123/deactivate \
  -H "Authorization: Bearer {accessToken}"
```
</RequestExample>

Deactivate a user account by ID.

<Note>
  Requires Admin role.
</Note>

### Path Parameters

<ParamField path="id" type="integer" required>
  User ID to deactivate
</ParamField>

### Response

Returns the deactivated user object.

### Error Codes

- `401` - Unauthorized or inactive admin account
- `404` - User not found
- `500` - Internal server error
