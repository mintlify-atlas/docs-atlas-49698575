## Log Song Play

<RequestExample>
```bash POST /api/v1/PlayHistory/LogSongHistory
curl -X POST https://api.swar.com/api/v1/PlayHistory/LogSongHistory \
  -H "Authorization: Bearer {accessToken}" \
  -H "Content-Type: application/json" \
  -d '"song123"'
```
</RequestExample>

Log a song play event in the user's listening history.

<Note>
  Requires authentication with Admin or User role. Call this endpoint whenever a user plays a song.
</Note>

### Request Body

The request body should be a JSON string containing the song ID.

<ParamField body="songId" type="string" required>
  The ID of the song that was played
</ParamField>

### Response

Returns a success message.

<ResponseExample>
```json Response
{
  "message": "Song history logged successfully."
}
```
</ResponseExample>

### Error Codes

- `401` - Inactive account
- `404` - User or song not found
- `500` - Internal server error

## Get User Play History

<RequestExample>
```bash GET /api/v1/PlayHistory/GetSongHistoryByUser
curl -X GET https://api.swar.com/api/v1/PlayHistory/GetSongHistoryByUser?unique=true \
  -H "Authorization: Bearer {accessToken}"
```
</RequestExample>

Retrieve the play history for the authenticated user.

<Note>
  Requires authentication with Admin or User role.
</Note>

### Query Parameters

<ParamField query="unique" type="boolean" default="true">
  If true, returns only unique songs (removing duplicates). If false, returns all play events including repeats.
</ParamField>

### Response

Returns an array of song IDs or play history objects based on the unique parameter.

<ResponseExample>
```json Response (unique=true)
[
  "song123",
  "song456",
  "song789"
]
```

```json Response (unique=false)
[
  {
    "songId": "song123",
    "playedAt": "2026-03-03T10:30:00Z"
  },
  {
    "songId": "song456",
    "playedAt": "2026-03-03T10:25:00Z"
  },
  {
    "songId": "song123",
    "playedAt": "2026-03-03T09:15:00Z"
  }
]
```
</ResponseExample>

### Error Codes

- `401` - Inactive account
- `404` - No play history found
- `500` - Internal server error

## Get All User History

<RequestExample>
```bash GET /api/v1/PlayHistory/GetAllUserHistory
curl -X GET https://api.swar.com/api/v1/PlayHistory/GetAllUserHistory \
  -H "Authorization: Bearer {accessToken}"
```
</RequestExample>

Retrieve play history for all users in the system.

<Note>
  Requires Admin role. Useful for analytics and reporting.
</Note>

### Response

Returns an array of play history objects for all users.

<ResponseExample>
```json Response
[
  {
    "userId": 123,
    "songId": "song123",
    "playedAt": "2026-03-03T10:30:00Z"
  },
  {
    "userId": 124,
    "songId": "song456",
    "playedAt": "2026-03-03T10:25:00Z"
  },
  {
    "userId": 123,
    "songId": "song789",
    "playedAt": "2026-03-03T09:15:00Z"
  }
]
```
</ResponseExample>

### Error Codes

- `404` - No play history found
- `500` - Internal server error
