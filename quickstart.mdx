# Quickstart Guide

This guide will walk you through setting up your Swar account and experiencing the core features of the platform.

## Prerequisites

Before you begin, ensure you have:

- A valid email address
- A secure password (minimum 8 characters)
- Access to the Swar API base URL

<Note>
All code examples use `https://api.swar.example.com` as the base URL. Replace this with your actual Swar API endpoint.
</Note>

## Step 1: Create Your Account

<Steps>
  <Step title="Register a new user">
    Create your Swar account by sending a POST request to the registration endpoint.

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api.swar.example.com/api/v1/users/register \
      -H "Content-Type: application/json" \
      -d '{
        "name": "John Doe",
        "email": "john.doe@example.com",
        "password": "SecurePass123!"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch('https://api.swar.example.com/api/v1/users/register', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        name: 'John Doe',
        email: 'john.doe@example.com',
        password: 'SecurePass123!'
      })
    });

    const user = await response.json();
    console.log('Registered user:', user);
    ```

    ```python Python
    import requests

    response = requests.post(
        'https://api.swar.example.com/api/v1/users/register',
        json={
            'name': 'John Doe',
            'email': 'john.doe@example.com',
            'password': 'SecurePass123!'
        }
    )

    user = response.json()
    print('Registered user:', user)
    ```
    </CodeGroup>

    **Request Body** (UserRegisterDTO):
    ```json
    {
      "name": "string",        // Required, max 100 characters
      "email": "string",       // Required, valid email format, max 255 characters
      "password": "string",    // Optional for external auth
      "externalId": "string"   // Optional, for OAuth integrations
    }
    ```

    <Warning>
    Password must be at least 8 characters long and meet security requirements.
    </Warning>
  </Step>

  <Step title="Verify registration">
    Upon successful registration, you'll receive a response containing your user details:

    ```json Response (RegisteredUserDTO)
    {
      "userId": 1,
      "name": "John Doe",
      "email": "john.doe@example.com",
      "role": "User",
      "status": "Active"
    }
    ```
  </Step>
</Steps>

## Step 2: Login to Your Account

<Steps>
  <Step title="Authenticate">
    Login with your credentials to receive authentication tokens.

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api.swar.example.com/api/v1/users/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "john.doe@example.com",
        "password": "SecurePass123!"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch('https://api.swar.example.com/api/v1/users/login', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        email: 'john.doe@example.com',
        password: 'SecurePass123!'
      })
    });

    const loginResult = await response.json();
    // Store tokens securely
    localStorage.setItem('accessToken', loginResult.accessToken);
    localStorage.setItem('refreshToken', loginResult.refreshToken);
    ```

    ```python Python
    response = requests.post(
        'https://api.swar.example.com/api/v1/users/login',
        json={
            'email': 'john.doe@example.com',
            'password': 'SecurePass123!'
        }
    )

    login_result = response.json()
    access_token = login_result['accessToken']
    refresh_token = login_result['refreshToken']
    ```
    </CodeGroup>

    **Request Body** (UserLoginDTO):
    ```json
    {
      "email": "string",     // Required, valid email format
      "password": "string"   // Required, 8-100 characters
    }
    ```
  </Step>

  <Step title="Store your tokens">
    The login response contains your authentication tokens:

    ```json Response (LoginResultDTO)
    {
      "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "userId": 1,
        "name": "John Doe",
        "email": "john.doe@example.com",
        "role": "User"
      }
    }
    ```

    <Note>
    Store the `accessToken` for API requests and the `refreshToken` for obtaining new access tokens when they expire.
    </Note>
  </Step>
</Steps>

## Step 3: Search for Songs

Now that you're authenticated, let's find some music!

<CodeGroup>
```bash cURL
curl -X GET "https://api.swar.example.com/api/v1/SongsData/GetSongsByQuery?query=bohemian%20rhapsody&songData=true&lyrics=false" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const accessToken = localStorage.getItem('accessToken');

const response = await fetch(
  'https://api.swar.example.com/api/v1/SongsData/GetSongsByQuery?query=bohemian+rhapsody&songData=true&lyrics=false',
  {
    headers: {
      'Authorization': `Bearer ${accessToken}`
    }
  }
);

const songs = await response.json();
console.log('Search results:', songs);
```

```python Python
import requests

response = requests.get(
    'https://api.swar.example.com/api/v1/SongsData/GetSongsByQuery',
    params={
        'query': 'bohemian rhapsody',
        'songData': True,
        'lyrics': False
    },
    headers={
        'Authorization': f'Bearer {access_token}'
    }
)

songs = response.json()
print('Search results:', songs)
```
</CodeGroup>

**Query Parameters:**
- `query` (string): Search term for songs
- `songData` (boolean): Include song metadata (default: true)
- `lyrics` (boolean): Include lyrics in results (default: false)

## Step 4: Create Your First Playlist

<Steps>
  <Step title="Create a playlist">
    Build your first custom playlist:

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api.swar.example.com/api/v1/Playlist/CreatePlaylist \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "playlistName": "My Favorites",
        "description": "My all-time favorite songs",
        "isPrivate": false
      }'
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://api.swar.example.com/api/v1/Playlist/CreatePlaylist',
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${accessToken}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          playlistName: 'My Favorites',
          description: 'My all-time favorite songs',
          isPrivate: false
        })
      }
    );

    const playlist = await response.json();
    console.log('Created playlist:', playlist);
    ```

    ```python Python
    response = requests.post(
        'https://api.swar.example.com/api/v1/Playlist/CreatePlaylist',
        headers={'Authorization': f'Bearer {access_token}'},
        json={
            'playlistName': 'My Favorites',
            'description': 'My all-time favorite songs',
            'isPrivate': False
        }
    )

    playlist = response.json()
    print('Created playlist:', playlist)
    ```
    </CodeGroup>

    **Request Body** (AddPlaylistDTO):
    ```json
    {
      "playlistName": "string",  // Required, 1-100 characters
      "description": "string",   // Optional
      "isPrivate": boolean        // Required
    }
    ```

    <Note>
    Users can create up to 3 playlists. Each playlist can contain up to 5 songs.
    </Note>
  </Step>

  <Step title="Add songs to your playlist">
    Add songs you found in the search to your new playlist:

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api.swar.example.com/api/v1/PlaylistSongs/AddSongToPlaylist \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "playlistId": 1,
        "songId": "song_xyz123"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://api.swar.example.com/api/v1/PlaylistSongs/AddSongToPlaylist',
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${accessToken}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          playlistId: 1,
          songId: 'song_xyz123'
        })
      }
    );

    const result = await response.json();
    console.log('Song added:', result);
    ```

    ```python Python
    response = requests.post(
        'https://api.swar.example.com/api/v1/PlaylistSongs/AddSongToPlaylist',
        headers={'Authorization': f'Bearer {access_token}'},
        json={
            'playlistId': 1,
            'songId': 'song_xyz123'
        }
    )

    result = response.json()
    print('Song added:', result)
    ```
    </CodeGroup>

    **Request Body** (AddSongToPlaylistDTO):
    ```json
    {
      "playlistId": number,   // Required
      "songId": "string"      // Required, max 50 characters
    }
    ```
  </Step>
</Steps>

## Step 5: Start Playing Music

Once you have songs in your playlist or from search results, you can start playing:

<Steps>
  <Step title="Log play history">
    When a user plays a song, log it to their play history:

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api.swar.example.com/api/v1/PlayHistory/LogSongHistory \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
      -H "Content-Type: application/json" \
      -d '"song_xyz123"'
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://api.swar.example.com/api/v1/PlayHistory/LogSongHistory',
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${accessToken}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify('song_xyz123')
      }
    );

    const result = await response.json();
    console.log(result.message);
    ```

    ```python Python
    response = requests.post(
        'https://api.swar.example.com/api/v1/PlayHistory/LogSongHistory',
        headers={'Authorization': f'Bearer {access_token}'},
        json='song_xyz123'
    )

    result = response.json()
    print(result['message'])
    ```
    </CodeGroup>
  </Step>

  <Step title="Get recently played songs">
    Retrieve your listening history:

    <CodeGroup>
    ```bash cURL
    curl -X GET "https://api.swar.example.com/api/v1/PlayHistory/GetSongHistoryByUser?unique=true" \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://api.swar.example.com/api/v1/PlayHistory/GetSongHistoryByUser?unique=true',
      {
        headers: {
          'Authorization': `Bearer ${accessToken}`
        }
      }
    );

    const history = await response.json();
    console.log('Recently played:', history);
    ```

    ```python Python
    response = requests.get(
        'https://api.swar.example.com/api/v1/PlayHistory/GetSongHistoryByUser',
        params={'unique': True},
        headers={'Authorization': f'Bearer {access_token}'}
    )

    history = response.json()
    print('Recently played:', history)
    ```
    </CodeGroup>

    **Query Parameters:**
    - `unique` (boolean): Return only unique songs (default: true)
  </Step>
</Steps>

## Token Refresh

When your access token expires, refresh it using your refresh token:

```javascript JavaScript
const refreshToken = localStorage.getItem('refreshToken');

const response = await fetch(
  'https://api.swar.example.com/api/v1/users/refresh-token',
  {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${refreshToken}`
    }
  }
);

const { accessToken } = await response.json();
localStorage.setItem('accessToken', accessToken);
```

<Warning>
The refresh token endpoint uses a different authentication scheme. Make sure to send the refresh token in the Authorization header.
</Warning>
